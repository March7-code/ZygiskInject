# ZygiskFrida 注入失败诊断步骤

## 1. 检查配置文件位置和权限

```bash
# 检查配置文件是否存在
adb shell "su -c 'ls -la /data/adb/re.zyg.fri/config.json'"

# 检查 gadget 是否存在
adb shell "su -c 'ls -la /data/local/tmp/re.zyg.fri/libgadget.so'"

# 检查配置文件内容
adb shell "su -c 'cat /data/adb/re.zyg.fri/config.json'"

# 设置正确的权限（如果需要）
adb shell "su -c 'chmod 644 /data/adb/re.zyg.fri/config.json'"
adb shell "su -c 'chmod 755 /data/local/tmp/re.zyg.fri/libgadget.so'"
```

## 2. 检查 Zygisk 模块状态

```bash
# 检查模块是否安装
adb shell "su -c 'ls -la /data/adb/modules/zygiskfrida/'"

# 检查 Magisk 是否启用 Zygisk
adb shell "su -c 'magisk --sqlite \"SELECT * FROM settings WHERE key='zygisk'\"'"

# 或者通过文件检查
adb shell "su -c 'cat /data/adb/magisk.db' | grep zygisk"
```

## 3. 检查模块的 .so 文件

```bash
# 检查 zygisk 模块 so 是否存在
adb shell "su -c 'ls -la /data/adb/modules/zygiskfrida/zygisk/'"

# 应该看到类似这样的文件：
# arm64-v8a.so
# armeabi-v7a.so
```

## 4. 查看完整 logcat

```bash
# 方法1: 清除日志后重新启动应用
adb logcat -c
adb shell am force-stop com.darvin.security
adb logcat -v time | grep -E "(ZygiskFrida|zygisk|Zygisk)"

# 方法2: 查看所有日志（包括 liblog）
adb logcat -b all -v time | grep -i frida

# 方法3: 不过滤，查看应用启动时的所有日志
adb logcat -c
adb shell am start -n com.darvin.security/.MainActivity
adb logcat > logcat.txt
```

## 5. 测试配置文件解析

创建一个简单的测试配置：

```bash
# 创建一个最简配置进行测试
adb shell "su -c 'cat > /data/adb/re.zyg.fri/config.json << EOF
{
    \"targets\": [
        {
            \"app_name\" : \"com.darvin.security\",
            \"enabled\": true,
            \"start_up_delay_ms\": 0,
            \"injected_libraries\": [
                {
                    \"path\": \"/data/local/tmp/re.zyg.fri/libgadget.so\"
                }
            ]
        }
    ]
}
EOF'"
```

## 6. 检查是否有 SELinux 问题

```bash
# 检查 SELinux 状态
adb shell "su -c 'getenforce'"

# 如果是 Enforcing，可以临时设置为 Permissive 测试
adb shell "su -c 'setenforce 0'"

# 测试后记得改回来
adb shell "su -c 'setenforce 1'"
```

## 7. 验证应用包名是否正确

```bash
# 获取正在运行的应用的实际包名
adb shell "ps -A | grep darvin"

# 或者使用 dumpsys
adb shell "dumpsys window | grep mCurrentFocus"
```

## 8. 检查 Gadget 架构是否匹配

```bash
# 检查应用的架构
adb shell "su -c 'cat /proc/\$(pidof com.darvin.security)/maps | grep -E \"lib.*\\.so\" | head -5'"

# 检查 gadget 的架构
adb shell "su -c 'file /data/local/tmp/re.zyg.fri/libgadget.so'"

# 如果应用是 32 位，需要使用 32 位的 gadget
adb shell "su -c 'ls -la /data/local/tmp/re.zyg.fri/libgadget32.so'"
```

## 9. 重启测试

```bash
# 完全重启设备以确保 Zygisk 模块加载
adb reboot

# 等待重启后，查看模块状态
adb wait-for-device
adb shell "su -c 'ls -la /data/adb/modules/zygiskfrida/'"
```

## 10. 替代测试方法

如果使用 config.json 失败，尝试旧的简单配置方式：

```bash
# 使用 target_packages 方式
adb shell "su -c 'echo com.darvin.security > /data/adb/re.zyg.fri/target_packages'"

# 设置 injected_libraries
adb shell "su -c 'echo /data/local/tmp/re.zyg.fri/libgadget.so > /data/adb/re.zyg.fri/injected_libraries'"
```

## 检查清单

- [ ] 配置文件在正确位置 `/data/adb/re.zyg.fri/config.json`
- [ ] gadget 文件存在且可执行
- [ ] Zygisk 已在 Magisk 中启用
- [ ] 模块已安装且未被禁用
- [ ] 设备已重启
- [ ] 应用包名正确
- [ ] gadget 架构与应用匹配
- [ ] SELinux 不阻止加载
- [ ] logcat 使用正确的过滤参数

## 常见问题

### 问题1: 没有任何日志
**原因**: 可能 Zygisk 模块根本没有加载
**解决**: 检查 Magisk 中 Zygisk 是否启用，模块是否安装

### 问题2: 有日志但没有注入
**原因**: 配置文件解析失败或包名不匹配
**解决**: 检查 JSON 格式，确认包名

### 问题3: gadget 加载失败
**原因**: 架构不匹配或文件损坏
**解决**: 确认应用的 ABI，使用对应的 gadget

### 问题4: 临时出现然后消失
**原因**: SELinux 或其他安全策略阻止
**解决**: 临时 setenforce 0 测试
